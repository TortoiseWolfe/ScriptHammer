## Critical Finding: Title Position Pattern Inversion

### Issue

The Inspector is flagging SVGs at `x=960` as PATTERN_VIOLATION when `x=960` is the **CORRECT** centered position per spec.

| Pattern     | Count    | Assessment                      |
| ----------- | -------- | ------------------------------- |
| title x=700 | Majority | **WRONG** - Off-center          |
| title x=960 | Minority | **CORRECT** - Centered per spec |

### Evidence

From `005-security-hardening/01-security-ux-enhancements.issues.md`:

> "title_x_oddball | x=960 is CORRECT per spec! Majority at x=700 are WRONG."

### Recommendation

**ESCALATE TO INSPECTOR TERMINAL:**

1. Invert the pattern violation logic
2. Flag `x=700` as violations (needs REGENERATE)
3. Accept `x=960` as correct pattern
4. Batch regeneration needed for ~25 SVGs with wrong title position
