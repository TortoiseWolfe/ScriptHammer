### SO-01: User Story Order Mismatch

**Current annotation panel order:**
```
[1] Brute Force Prevention → points to lockout warning at BOTTOM of UI
[2] Email Validation → points to disposable email warning at TOP of UI
[3] Password Strength → points to password meter in MIDDLE of UI
[4] Error Recovery → points to "Forgot password?" link below form
```

**Visual reading flow (top to bottom):**
```
TOP:    Email field + disposable warning [currently callout 2]
MIDDLE: Password field + strength meter [currently callout 3]
BOTTOM: Lockout warning message [currently callout 1] ← OUT OF ORDER
BELOW:  Error recovery link [currently callout 4]
```

**Expected order (matching visual flow):**
```
[1] Email Validation (top of form)
[2] Password Strength (middle of form)
[3] Brute Force Prevention (bottom - lockout state)
[4] Error Recovery (recovery action)
```

**UX Rationale:**
- Users scan forms top-to-bottom
- Callout numbers should follow natural eye movement
- Current order (1 at bottom) creates cognitive dissonance
- Lockout warning appears AFTER failed attempts, so it's the END of the error flow, not the beginning

**Action Required:**
1. Technical Writer to reorder user stories in spec:
   - Move US-003 (Brute Force Prevention) from position 1 to position 3
   - Move US-007 (Email Validation) from position 2 to position 1
   - Move US-008 (Password Strength) from position 3 to position 2
2. After spec update, regenerate wireframe with corrected callout order

**Note:** This is a recurring pattern - see also 000-landing-page SO-01, 005-security-hardening_02
