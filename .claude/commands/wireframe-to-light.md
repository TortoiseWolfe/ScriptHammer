description: Convert dark theme SVG wireframes to light theme by replacing the color palette.
---

## User Input

```text
$ARGUMENTS
```

## Outline

Convert existing dark theme wireframes to light theme by applying color palette replacements in-place.

### 1. Select Wireframes to Convert

If `$ARGUMENTS` is provided, use it as file path(s) or category. Otherwise, show interactive selection.

**Selection options**:
- `all` - Convert all 85 UX/Frontend wireframes
- `foundation` - Convert Foundation category (000-006)
- `core` - Convert Core Features category (007-012)
- `auth` - Convert Auth OAuth category (013-016)
- `enhancements` - Convert Enhancements category (017-021)
- `integrations` - Convert Integrations category (022-026)
- `polish` - Convert Polish category (027-030)
- `payments` - Convert Payments category (038-043)
- `code-quality` - Convert Code Quality category (044-045)
- Specific file path(s)

**Reference WIREFRAME_INDEX.md** for the UX/Frontend wireframes list. Do NOT convert Backend/Abstract wireframes (they should stay dark theme).

Ask the user which wireframes to convert if no argument provided.

### 2. Confirm Selection

Before converting, show:
- Number of files to convert
- List of file paths
- Ask for confirmation to proceed

### 3. Apply Color Transformations

For each selected SVG file, apply these replacements **in order** (order matters to avoid double-replacement):

#### Step 1: Background Gradients (in `<defs>`)
```
stop-color:#0f172a  →  stop-color:#e0e7ff  (Indigo 100)
stop-color:#1e293b  →  stop-color:#c7d2fe  (Indigo 200)
```

#### Step 2: Panel Fills
```
fill="#1e293b"  →  fill="#fafafa"   (off-white panels)
fill="#334155"  →  fill="#f3f4f6"   (gray-100 secondary)
fill="#0f172a"  →  fill="#ffffff"   (white cards/inner areas)
```

#### Step 3: Border Strokes
```
stroke="#475569"  →  stroke="#a5b4fc"  (Indigo 300)
stroke="#334155"  →  stroke="#c7d2fe"  (Indigo 200)
```

#### Step 4: CSS Text Classes (in `<style>` block)
```
fill: #ffffff  →  fill: #1f2937
fill: #cbd5e1  →  fill: #374151
fill: #94a3b8  →  fill: #6b7280
fill: #64748b  →  fill: #9ca3af
```

#### Step 5: Inline Text Fills
```
fill="#fff"      →  fill="#1f2937"
fill="#ffffff"   →  fill="#1f2937"
fill="#cbd5e1"   →  fill="#374151"
fill="#94a3b8"   →  fill="#6b7280"
fill="#64748b"   →  fill="#9ca3af"
```

#### Step 6: Dot Grid Pattern
```
opacity="0.05"  →  opacity="0.4"
```
In the pattern dots, change:
```
fill="#fff"  →  fill="#a5b4fc"  (Indigo 300 - matches borders)
```

#### Colors to LEAVE UNCHANGED
These brand and status colors remain the same in both themes:
- Primary: `#8b5cf6` (violet)
- Accent: `#d946ef` (fuchsia)
- Accent gradient: `url(#accentGrad)`
- Success: `#22c55e`
- Warning: `#eab308`
- Error: `#ef4444`

### 4. Update Watermark

Add conversion info to the SVG header comment. Find:
```xml
<!-- ============================================================
     GENERATED BY: /wireframe skill
     SOURCE: [path]
     DATE: [date]
     DO NOT MANUALLY EDIT - Regenerate with /wireframe command
     ============================================================ -->
```

Replace with:
```xml
<!-- ============================================================
     GENERATED BY: /wireframe skill
     CONVERTED TO LIGHT: /wireframe-to-light [YYYY-MM-DD]
     SOURCE: [path]
     ORIGINAL DATE: [date]
     DO NOT MANUALLY EDIT - Regenerate with /wireframe-light command
     ============================================================ -->
```

### 5. Save and Report

1. Write the modified SVG back to the same file path
2. After processing all files, report:
   - Total files converted
   - List of converted file paths
   - Any files that had issues

### 6. Verification Reminder

Remind the user:
```
Conversion complete! Verify the results:
1. Open docs/design/wireframes/index.html in a browser
2. Navigate to converted wireframes
3. Check that colors display correctly on light backgrounds
```

## Example Usage

```bash
# Convert all UX/Frontend wireframes
/wireframe-to-light all

# Convert specific category
/wireframe-to-light foundation

# Convert specific file
/wireframe-to-light docs/design/wireframes/002-cookie-consent/01-consent-modal.svg
```
